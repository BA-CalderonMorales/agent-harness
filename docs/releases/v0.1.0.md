# Release v0.1.0 - Domain Bucket Architecture

## Overview

This release introduces the bucket pattern across all domains for consistent, maintainable architecture.

## Changes

### Domain Refactors (4 commits)

1. **tools**: ToolBucket interface with FileSystemBucket
   - `internal/tools/base.go` - ToolBase interface
   - `internal/tools/buckets/filesystem.go` - File operations
   - `internal/tools/defaults/` - Centralized config

2. **permissions**: PermissionBucket interface
   - `internal/permissions/base.go` - PermissionBase interface  
   - `internal/permissions/buckets/` - UserSettings, Mode buckets
   - `internal/permissions/defaults/` - Mode defaults

3. **agent**: AgentBucket interface
   - `internal/agent/base.go` - AgentBase interface
   - `internal/agent/buckets/` - Executor, Conversation buckets
   - `internal/agent/defaults/` - Default configs

4. **llm, state, contextmgr, config, tui, services**: Base interfaces
   - Each domain has `base.go` with Base interface
   - Orchestrator for routing
   - BucketCapabilities for metadata

## Pattern Applied

All domains now follow the bucket pattern from `internal/loop/`:

```go
// Base interface
type XBase interface {
    Name() string
    CanHandle(...) bool
    Execute(...) XResult
    Capabilities() XBucketCapabilities
}

// Orchestrator coordinates
type XOrchestrator struct {
    buckets []XBase
}

// Routing without knowing internals
func (o *XOrchestrator) Execute(...) XResult {
    bucket, _ := o.FindBucket(...)
    return bucket.Execute(...)
}
```

## Benefits

- **Clear debugging path**: Each bucket is isolated
- **Easy testing**: Mock buckets for unit tests
- **Predictable extension**: Add new buckets without changing orchestrator
- **No breaking changes**: All existing APIs remain functional

## Migration Guide

No migration needed. Existing code continues to work.

To use new bucket pattern:

```go
// Create buckets
fsBucket := buckets.FileSystem("/workspace")

// Create orchestrator
orch := tools.NewToolOrchestrator(fsBucket)

// Execute
result := orch.ExecuteTool(ctx)
```
