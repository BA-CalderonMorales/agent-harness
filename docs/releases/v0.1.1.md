# Release v0.1.1 - Layered Architecture Refactor

## Overview
Complete codebase restructure into clean layered architecture with proper dependency chains.

## Changes

### Architecture
- **Layer 0 (core)**: config, state - foundation with no internal deps
- **Layer 1 (runtime)**: llm, tools, permissions, services - core runtime
- **Layer 2 (agent)**: core/, conversation/, executor/ - agent orchestration
- **Layer 3 (session)**: loop/, contextmgr/ - session management
- **Layer 4 (interface)**: tui/, commands/, approval/ - user interface

### Rules Enforced
- Import only from lower layers
- Subdomains live inside parent folders
- No circular dependencies
- Chainable debug path: tui → agent → runtime/llm

### Code Organization
- Broke apart main.go into 6 focused files
- Removed dead code (conversation.go, conversation_test.go)
- Removed empty directories
- Fixed layer violation (agent no longer imports ui)

## Migration
No breaking changes. Internal refactor only.
